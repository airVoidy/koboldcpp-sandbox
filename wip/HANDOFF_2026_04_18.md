## Core architectural closures reached today

**Atomic layer = RAM-level structural primitive, not semantic layer**:
- **Atom** = one assembly-typed RAM object; fixed after placement; name scope-local
- **Atomic L0** instruction set = MOV + CALL + batch. Nothing else. One object, one action
- Atomic layer **waits for external actors** (Lexical / Jupyter / our exec / etc.) to do work, then **wraps their trace** into composable structures
- **Cannot mix** atomic with non-atomic layers — uniform approach or nothing
- Minimal primitives = **object + directed list** (Kolmogorov/Shannon floor)

**Identity discipline**:
- **Hash = global identity**, cross-language, stateless, handshake-free
- **Name = placeholder-on-self within (scope, projection)** — scope-local perspective, not identity
- Agreement-before-emergence: rule pre-declared, hash computed at object-appearance automatically
- Ref-link itself = atomic object in projection form. Self-ref anchor (@a : @a) = bind attachment site

**Bind mechanism** (at jne+mov at self-ref anchor):
- Declaration moment = log entry append = bind attach = first execution (single event, performative naming)
- `.value = X` → `value(obj, X)` via UFCS — method invocation, not property assignment
- Baked cost = **0 ops** at runtime (direct memory read from baked slot)
- The "unbaked" 2-op floor (jne+mov) only exists pre-compile

**Loop-with-single-exit architecture**:
- The ONLY branch in assembly: `jne @a +1` that doesn't fire → fall-through = loop exit
- Loop body = accumulate patches (rows appended, MOV per patch)
- Fall-through = commit accumulated payload as new state (single atomic swap)
- **Virtual** ops — not literally executed, structural position markers
- Entry point = quiescent state where 1 MOV completes any update

**Cross-runtime federation**:
- Runtime (VM / interpreter) — per-side native, not shared
- Commands named identically by convention
- Assembly batch structurally identical (MOV/CALL pattern)
- Serialization bridges boundaries via text/JSON
- Static transform representation **runtime-independent**

**Triple waypoints at entry point** (single address, polymorphic):
- Full object → full object (replace, 1 MOV)
- Apply patch → full object (targeted MOV at indexed field)
- Full object output (read, 1 MOV)
- Input shape dispatches; output always resolved state

**Immediate projection invariant**:
- "Если мы смогли собрать строку на отправку → все invariants уже спроектились локально"
- Line existence = evidence of local resolution
- Server round-trip = promise with timestamp, doesn't gate rendering
- Any UI wait even 1 tick between line-assembly and projection-update = architectural bug
